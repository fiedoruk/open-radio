## First write

- Complete private 16 MiB factory backup and checksum: verified before write.
- Full software gate and two deterministic clean candidate builds: `PASS`.

- The device reset after upload and entered a repeating wake/blank cycle:
  `H1 BLOCKED`.

## Diagnosis

- Serial evidence: loop-task stack canary panic during
  `PreferencesFavoriteRepository::begin()`.
- Root cause: two local favorite-slot values each held 32 entries with
  193-byte titles. Their combined stack footprint exceeded the Arduino loop
  task stack during initialization.
- The same class also created a full local rollback copy during favorite
  updates and removals, leaving a second latent stack-overflow path.

## Correction candidate

- Slot selection now uses small metadata values and the existing static NVS
  scratch buffer; full slot arrays are no longer allocated on the loop stack.
- Favorite rollback storage is now a repository member, not a local stack copy.
- A focused source regression test rejects both unsafe allocation patterns.
- Guarded upload and monitor output now redact device identifiers and port
  paths.
- The first hotfix reflash removed the observed reset loop, but physical display
  inspection found incorrect colors and low-contrast text: `H1 BLOCKED`.
- Root cause: the renderer owns native RGB565 values while M5GFX defaults a raw
  `uint16_t` image pointer to swapped-byte input. The board UI now calls
  `setSwapBytes(true)` before its first framebuffer upload, and a focused source
  check locks that ordering before `pushImage`.
- The owner confirmed the corrected display colors, legibility and local audio
  on the third guarded write. The reset loop did not return.
- The next UX candidate adds a 184 px setup QR, projects the physical A/B/C
  zones through the same screen action dispatcher and removes LCD transport
  buttons from the minimal home.
- The minimal home reads date and time from the Core2 RTC. The stabilization
  candidate sets that RTC from optional SNTP and binds cached, failure-isolated
  weather only after secured Wi-Fi and a valid network clock exist.
- UX candidate static RAM: `122,668` bytes.
- UX candidate application flash: `2,223,481` bytes.
- UX candidate firmware binary: `2,230,064` bytes, below the 2.25 MiB gate.
