# Governance

## Model

The project starts as maintainer-led. The founder owns the roadmap, releases,
security decisions and project trademark.

Founder and original author: Tomasz Fiedoruk, https://fiedoruk.pl.

## Decisions

- Architecture changes use ADRs.
- Large product changes use public RFCs after the repository launch.
- Normal pull requests require one maintainer review.
- Security, manual USB rescue, network and stable station-catalog changes require two reviews.
- A contributor becomes a reviewer or maintainer through sustained, documented work.

## No hidden authority

Roles, release permissions and decision changes are recorded in this file.
Private infrastructure does not become the source of truth for public firmware.
